## Agent rules

- Never use Python for ad hoc scripting, file edits, or JSON/text munging. Use `bun` for scripts that need a real language; otherwise use shell tools (`rg`, `sed`, `awk`, `jq`), repo-native commands, or `apply_patch`.
- The AI agent fleet (per-account `claude-<name>` / `codex-<name>` wrappers + their settings/memory/skills/hooks) is managed by **`kfleet`**. Home Manager owns the source assets in this repo under `kfleet/` and links them into `~/.kfleet/`; edit `kfleet/config.yaml`, `kfleet/CLAUDE.md`, `kfleet/CLAUDE.auto.md`, `kfleet/templates/`, `kfleet/skills/`, or `kfleet/skills-codex/`, then run `hms` (which runs `kfleet apply`) or run `kfleet apply` for asset-only refreshes. Do NOT edit generated homes like `~/.claude-<name>/`, `~/.codex-<name>/`, or generated wrappers under `~/.kfleet/bin/`. `modules/agent-config` is deprecated legacy seed material; do not use it as the source of truth. Only `multi-gh`/`multi-gws` accounts are still Nix-managed in `home-template.nix`.
- **kteam is EXPERIMENTAL.** Whenever you drive work through `kteam`, ALWAYS spin up a babysitter subagent (opus, sonnet, or luna) alongside the team. Its job: watch the sessions (`kteam status`, `kteam snapshot`, transcripts under `~/.kteam/<id>/`), capture every malfunction or stall, debug it, guess the likely source in `modules/kteam-ts`, and APPEND the finding to `kteam-prob.md` at the repo root (problem, evidence, suspected code path, workaround). This applies on every machine â€” the Mac and the boxes; commit/push the log so entries merge. Do not silently work around a kteam bug without logging it.
- **Secrets (SOPS + age).** `secrets.enc.yaml` (committed, encrypted) pairs with `secrets.yaml` (gitignored, decrypted working copy). ALWAYS work on the decrypted `secrets.yaml` â€” never `sops edit` the enc file (that desyncs the pair). Rules:
  - Decrypt ONLY if `secrets.yaml` doesn't exist: `./scripts/secrets/decrypt.sh` (it refuses to overwrite an existing working copy).
  - Re-encrypt ONLY before committing: `./scripts/secrets/encrypt.sh` (skips when already in step, so no ciphertext churn). Then `git add secrets.enc.yaml`.
  - The `a-secrets-sync` pre-commit hook runs on EVERY commit and blocks it when `secrets.yaml` has edits not yet in `secrets.enc.yaml` â€” if a commit fails there, run encrypt and retry. Never delete or `--no-verify` around it.
  - Never commit `secrets.yaml` itself.
- **Box provisioning.** `pls box:up -- <aws|digitalocean|oci> [--replicate]` provisions the box via OpenTofu (`infra/<cloud>/`, all three clouds are identical: x86_64 Ubuntu 24.04 + shared `infra/cloud-init.yaml.tftpl`); `pls box:down -- <cloud>` destroys it. Cloud creds live in sops at `.box.clouds.<cloud>` in `secrets.yaml`; tofu state is local + gitignored, so down must run on the machine that ran up. The **userdata bootstraps the system layer on first boot**: Nix (+ daemon trusted-user), first home-manager switch, tailscaled, systemd lingering â€” a bare `box:up` already converges the environment (secrets skipped until the age key arrives; `load-secrets` no-ops without it).
- **The box has DIVERGED (2026-07-13).** The provisioned box (kirin-box) is now an independent machine: do NOT run `box:replicate`, rsync, or box-side `git pull` on the user's behalf unless explicitly asked. In particular `replicate`'s loose-dir sync uses `rsync --delete` (Macâ†’box mirror) and WOULD DESTROY box-side changes under `~/Workspace/atomi/{nitroso,alcohol,diene,actions}`. Replicate remains the tool for provisioning FRESH boxes.
- **Box replication = age key + Workspace, nothing else.** `./scripts/box/replicate.sh <user@host> [profile]` (or `pls box:replicate -- ...`) seeds the local age key, re-runs `home-manager switch` (materializes secrets; falls back to full `scripts/setup.sh` on boxes without the cloud-init bootstrap), and runs `scripts/box/clone-stuff.sh` to clone the sops-encrypted manifest (`.box.repos` in `secrets.yaml`) into `~/Workspace`. Everything else travels via git + home-manager (packages, zsh, terminfo entries in `modules/terminfo/`, nightly restic backup timer â€” declared in `home-template.nix`, Linux-gated). Regenerate the repo manifest with `./scripts/box/gen-repos.sh` (it stays inside sops on purpose â€” this repo is public and the list names private/work repos), then re-encrypt.
- **When a command needs `sudo`** (e.g. `hms`, which runs `sudo darwin-rebuild`), the Bash tool has no tty to type a password into. Don't hand it back â€” give `sudo` an **askpass helper** that pops a macOS `osascript` dialog, and run with `sudo -A` (run the build as a background job since it takes minutes). The dialog must state the **agent** (claude/codex + name), **repo**, **folder path**, **worktree**, and **purpose**. Do NOT use the `sudo -S -v` timestamp-prime trick â€” `tty_tickets` defeats it. Full pattern in `kfleet/CLAUDE.md` â†’ "Elevated commands (sudo via osascript askpass)".
